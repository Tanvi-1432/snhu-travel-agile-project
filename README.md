# SNHU Travel Agile Project

This project demonstrates the application of Agile and Scrum methodologies throughout the software development life cycle (SDLC). The project was completed as part of CS-250 and focuses on building and iterating on features for the SNHU Travel application.

## Project Overview

The goal of this project was to simulate a real-world Agile environment by working through different Scrum roles, including Product Owner, Developer, Tester, and Scrum Master. The project involved creating user stories, developing features, writing test cases, and adapting to changing requirements.

A key example from this project is the transition from a general travel application to a detox and wellness-focused travel experience. This change was handled using Agile principles, allowing updates to be made without restarting development.

## Key Concepts Demonstrated

- Agile and Scrum methodology  
- Software Development Life Cycle (SDLC) in an Agile environment  
- User stories and acceptance criteria  
- Sprint planning and iterative development  
- Test case creation and validation  
- Handling changing requirements  
- Sprint review and retrospective analysis

## Reflection

### How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?
I interpret user needs by breaking them down into clear, simple requirements that focus on what the user is trying to achieve. Instead of focusing only on features, I focus on the outcome the user wants. User stories help with this by structuring requirements in a way that keeps the focus on the user’s goal. They also include acceptance criteria, which makes it clear when a feature is complete.

### How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?
I approach development by breaking problems into smaller parts and working step by step. I focus on building a working version first, then improving it over time. Agile supports this approach by allowing continuous feedback and iteration. In the future, I want to continue using user stories, iterative development, and regular check-ins to stay aligned and improve efficiency.

### What does it mean to be a good team member in software development?
Being a good team member means communicating clearly, staying accountable, and being open to feedback. It also means understanding your role while supporting the team when needed. In Agile, collaboration is important because the team depends on shared progress and communication to succeed.
